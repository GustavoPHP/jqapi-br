jQuery.noConflict()

Categories: Core | Miscellaneous > Setup Methods

jQuery.noConflict( [ removeAll ] ) Retorna: Object

Descrição: Faz o jQuery abrir mão do controle da variável.

  • adicionado na versão: 1.0jQuery.noConflict( [ removeAll ] )

    removeAllUm booleano que aponta para todas as variáveis do jQuery a remoção do escopo global(incluindo o jQuery em si).

Muitas bibliotecas JavaScript usam $ como nome de função ou variável, do mesmo jeito que o jQuery faz. No caso do jQuery, $ é apenas um atalho para jQuery, logo todas as funcionalidades estão disponíveis sem usar $. Se precisamos usar outra biblioteca JavaScript junto do jQuery, nós podemos retornar controle devolta de $ para outra biblioteca com uma chamada $.noConflict():

<script type="text/javascript" src="other_lib.js"></script>
<script type="text/javascript" src="jquery.js"></script>
<script type="text/javascript">
  $.noConflict();
  // Code that uses other library's $ can follow here.
</script>

Esta técnica é especialmente efetiva em conjunção com o método .ready() habilitado para o atalho do objeto jQuery, como no retorno passado por .ready() nós podemos usar $ se desejarmos ficar destemidos em relação a conflitos no futuro:

<script type="text/javascript" src="other_lib.js"></script>
<script type="text/javascript" src="jquery.js"></script>
<script type="text/javascript">
  $.noConflict();
  jQuery(document).ready(function($) {
    // Code that uses jQuery's $ can follow here.
  });
  // Code that uses other library's $ can follow here.
</script>

Se necessário, podemos liberar o nome jQuery, passando assim true como um argumento para o método. Isso raramente é necessário, e deve-se fazer isso(por exemplo, se precisamos usar várias versões da biblioteca jQuery na mesma página), é preciso considerar que a maioria dos plugins dependem da presença da variável jQuery e podem não funcionar corretamente nesta situação.

Exemplos:

Exemplo: Mapeia o objeto original que era referênciado por $ de volta para $. 
jQuery.noConflict();
// Do something with jQuery
jQuery("div p").hide();
// Do something with another library's $()
$("content").style.display = 'none';

Exemplo: Inverte o atalho $ e então cria e executa a função para prover o $ como um atalho jQuery dentro do escopo da função. Dentro da função o objeto original $ não está disponível. Isso funciona bem para a maioria dos plugins que não dependem de qualquer outra biblioteca. 

jQuery.noConflict();
(function($) { 
  $(function() {
    // more code using $ as alias to jQuery
  });
})(jQuery);
// other code using $ as an alias to the other library

Exemplo: Você pode atrelar o jQuery.noConflict () com a abreviação pronto para um código compacto. 

jQuery.noConflict()(function(){
    // code using jQuery
}); 
// other code using $ as an alias to the other library

Exemplo: Cria um atalho diferente ao invés de jQuery para usar no resto do script. 

var j = jQuery.noConflict();
// Do something with jQuery
j("div p").hide();
// Do something with another library's $()
$("content").style.display = 'none';

Exemplo: Move completamente o jQuery para um novo namespace em outro objeto. 

var dom = {};
dom.query = jQuery.noConflict(true);

Result:

// Do something with the new jQuery
dom.query("div p").hide();
// Do something with another library's $()
$("content").style.display = 'none';
// Do something with another version of jQuery
jQuery("div > p").hide();
