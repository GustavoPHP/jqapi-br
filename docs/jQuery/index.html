Na primeira formulação acima, jQuery() — que também pode ser escrita como $() — busca por meio do DOM por quaisquer elementos que correspondam ao seletor fornecido e cria um novo objeto jQuery que faz referência a estes elementos:

$('div.foo');

Selector Context

Por padrão, seletores executam suas pesquisas dentro da DOM a partir da raiz do documento. No entanto, um contexto alternativo pode ser dada para a busca usando o segundo parâmetro opcional para $() função. Por exemplo, para fazer uma pesquisa dentro de um manipulador de eventos, a busca pode ser restringido da seguinte forma::

$('div.foo').click(function() {
  $('span', this).addClass('bar');
});

Quando a pesquisa para o selector de extensão é restrita ao contexto de this, abrange somente o elemento clicado interno vai receber a classe adicional.

Internamente, o contexto de seleção é implementado com o método .find() , então $('span', this) equivale a $(this).find('span').

Usando elementos DOM

A segunda e terceira formulações desta função cria um objeto jQuery usando um ou mais elementos DOM que já foram selecionados em alguma outra forma. Um uso comum desta funcionalidade é chamar métodos jQuery em um elemento que tem sido passado para uma função callback através da palavra-chave this:

$('div.foo').click(function() {
  $(this).slideUp();
});

Este exemplo faz com que elementos que devem ser escondidas com uma animação de slide quando clicado. Porque o manipulador recebe o item clicado na palavra-chave this como um elemento DOM puro, o elemento deve ser passado para a função $() antes da aplicação de métodos jQuery para isso.

Dados XML retornados de uma chamada Ajax podem ser passados ​​para a função $() como elementos individuais de XML estruturado podendo ser recuperados usando .find() e outros métodos transversais DOM.

$.post('url.xml', function(data) {
  var $child = $(data).find('child');
})

Clonando Objetos jQuery

Returning an Empty Set

Quando um objeto jQuery é passado para a função $(), é criado um clone desse objeto. Esse novo objeto jQuery faz referência aos mesmos elementos DOM do objeto original.

Como parte do jQuery 1.4, chamando o método jQuery() com no arguments(vazio) retorna um conjunto vazio jQuery com um .length de propriedade 0). Nas versões anteriores do jQuery, Isto deveria retornar um conjunto contendo a raiz do documento.

Exemplos:

Exemplo: Encontrar todos elementos p que são filho do elemento div e aplicar uma borda a eles. 

<!DOCTYPE html>
<html>
<head>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p>one</p> <div><p>two</p></div> <p>three</p>
<script>
  $("div > p").css("border", "1px solid gray");
</script>

</body>
</html>

Demonstração:

Exemplo: Encontre todos os inputs do tipo radio dentro do primeiro formulário do documento. 

$("input:radio", document.forms[0]);

Exemplo: Encontre todos os elementos div dentro de um documento XML através de uma resposta AJAX. 

$("div", xml.responseXML);

Exemplo: Aplicar preto na cor de fundo da página. 

$(document.body).css( "background", "black" );

Exemplo: Esconder todos elementos input dentro de um formulário. 

$(myForm.elements).hide()

Criando Novos Elementos

Se uma string é passada como parâmetro $(), o jQuery examina a string para ver se se parece com HTML(i.e., Contém <tag ... > em algum lugar na string). Se não, a string é interpretada como uma expressão de seletor, como explicado abaixo. Mas se a string parece ser um fragmento de HTML, o jQuery tenta criar um novo elemento DOM como descrito pelo HTML. Então um objeto jQuery é criado e retornado a referência para estes elementos. Você pode realizar qualquer um dos métodos usuáis nesse objeto:

$('<p id="test">My <em>new</em> text</p>').appendTo('body');

Quando o HTML é mais complexo que uma simples tag sem atributos, como visto no exemplo acima, a criação real dos elementos é tratada pelo mecanismo do navegador innerHTML. Na maioria dos casos, jQuery cria um novo elemento <div> e configura a propriedade innerHTML do elemento para o fragmento de HTMLque foi passado. Quando o parâmetro tem uma tag simples, como $('<img />') ou $('<a></a>'), o jQuery cria o elemento usando JavaScript nativo, funçãocreateElement().

Quando passar um HTML complexo, alguns navegadores podem não gerar um DOM que exatamente repete o código HTML fornecido. Por exemplo, Internet Explorer anterior a versão 8 irá converter todas propriedades href de links para URLS absolutas, e Internet Explorer anterior a versão 9 não irá manusear corretamente elementos de HTML5 sem adicionar separado camada de compatibilidade.

Para garantir compatibilidade de plataformas, o fragmento tem que ser completo. Tag que cotenham outros elementos deveriam ter a tag fechada:

$('<a href="http://jquery.com"></a>');

Alternativamente, jQuery habilita a sintaxe da tag XML-like (com ou sem espaço antes da barra):

$('<a/>');

Tags que não podem conter elementos podem ser rapidamente fechadas ou não:

$('<img />');
$('<input>');

Como parte do jQuery 1.4, o segundo argumento pode aceitar um mapa constituido de um super-conjunto de propriedades que podem ser passados para o método .attr(). Além disso, qualquer tipo de evento pode ser passado, e seguindo os métodos jQuery podem ser chamados: val, css, html, text, data, width, height, ou offset. Perceba que o Internet Explorer não habilitará você a criar um elemento input ou button e mudar o tipo; você precisa especificar o tipo usando '<input type="checkbox" />' por exemplo.

Exemplos:

Exemplo: Cria um elemento div (e todo seu conteúdo) dinamicamente e o anexa ao elemento body.Internamente, um elemento é criado e sua propriedade innerHTML é definida pela marcação dada. 

$("<div><p>Hello</p></div>").appendTo("body")

Exemplo: Cria alguns elementos DOM. 

$("<div/>", {
  "class": "test",
  text: "Click me!",
  click: function(){
    $(this).toggleClass("test");
  }
}).appendTo("body");

jQuery( callback ) Retorna: jQuery

Descrição: Liga uma função para ser executada quando o DOM terminou de carregar.

  • adicionado na versão: 1.0jQuery( callback )

    callbackA função a ser executada quando o DOM estiver carregado.

Esta função se comporta exatamente como $(document).ready(), na medida em que deve ser usado para envolver outras operações $() na sua página que depende do DOM estar pronto. Enquanto essa função é, tecnicamente, em cadeia, não há realmente muito uso para o encadeamento contra ela.

Exemplos:

Exemplo: Executa a função quando o DOM está pronto para ser usado. 

$(function(){
   // Document is ready
 });

Exemplo: Use ambos atalhos para $(document).ready() e o argumento para escrever código jQuery à prova de falhas usando o atalho $ sem depender o atalho global. 

jQuery(function($) {
    // Your code using failsafe $ alias here...
  });
