event.stopImmediatePropagation()

Categorias: Events > Event Object

event.stopImmediatePropagation()

Descrição: Mantém o resto dos manipuladores sendo executados e impede que o evento interfira através a árvore DOM.

  • adicionado na versão: 1.3event.stopImmediatePropagation()

Além de manter todos os manipuladores adicionais sobre um elemento de serem executados, este método também impede que interfira implicitamente chamando event.stopPropagation(). Para simplesmente prevenir o evento da chamada aos elementos antecessores mas permitir outros manipuladores de eventos de executar no mesmo elemento, nós podemos utilizar event.stopPropagation().

Use event.isImmediatePropagationStopped() para saber se o métodos foi chamado alguma vez (neste objeto de evento).

Additional Notes:

  • Enquanto o .live() manipula os métodos de eventos uma vez que foi propagado no topo do documento, isso não possibilita parar a propagação de elementos live. Similarmente, eventos chamados por .delegate() sempre irão propagar ao elemento que eles foram delegados; manipuladores de eventos em qualquer elemento abaixo, isso irá ser executado pelo tempo que o manipulador de evento for chamado.

Exemplo:

Previne outro manipulador de evento de ser chamado.

<!DOCTYPE html>
<html>
<head>
  <style>
p { height: 30px; width: 150px; background-color: #ccf; }
div {height: 30px; width: 150px; background-color: #cfc; }
</style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p>paragraph</p>
<div>division</div>
<script>
$("p").click(function(event){
  event.stopImmediatePropagation();
});
$("p").click(function(event){
  // This function won't be executed
  $(this).css("background-color", "#f00");
});  
$("div").click(function(event) {
  // This function will be executed
    $(this).css("background-color", "#f00");
});</script>

</body>
</html>
