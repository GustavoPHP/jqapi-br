.submit()

Categories: Events > Form Events | Forms

.submit( handler(eventObject) ) Retorna: jQuery

Descrição: Atrela um manipulador de evento ao evento JavaScript "submit", ou dispara o evento em um elemento.

  • criado na versão: 1.0 .submit( handler(eventObject) )

    handler(eventObject) Função a ser executada toda vez que o evento ocorre.

  • criado na versão: 1.4.3 .submit( [ eventData ], handler(eventObject) )

    eventData Mapa de dados a serem passados ao manipulador do evento.

    handler(eventObject) Função a ser executada toda vez que o evento ocorre.

  • criado na versão: 1.0 .submit()

Nos dois primeiros casos mostrados anteriormente este método é uma forma abreviada para o método .bind('submit', handler) e no terceiro para o método .trigger('submit').

O evento submit ocorre quando o usuário envia um formulário. Este evento destina-se a ser atrelado a elementos <form>. Formulários são enviados por um clique em um elemento <input type="submit">, <input type="image"> ou <button type="submit"> ou ainda, por acionamento da tecla Enter quando determinados elementos do formulário têm o foco.

Dependendo do navegador, acionar a tecla Enter, envia o formulário somente se ele tiver apenas um campo de texto ou quando há um botão de envio.

Considere a seguinte marcação HTML:

<form id="target" action="destination.html">
  <input type="text" value="Ei você" />
  <input type="submit" value="Go" />
</form>
<div id="other">
  Dispara o evento
</div>

O evento pode ser atrelado ao formulário:

$('#target').submit(function() {
  alert('Chamado o manipulador do evento .submit()');
  return false;
});

Quando o formulário for enviado o alerta é mostrado. O alerta acontece antes do envio do formulário. É possível cancelar o envio com uso do método jQuery .preventDefault() ou retornando false na função manipuladora do evento. Podemos ainda, disparar o evento quando um outro elemento for clicado:

$('#other').click(function() {
  $('#target').submit();
});

Segundo este código disparando o evento o alerta é mostrado. Ainda mais, a ação submit será disparada e o formulário será enviado.

A partir da jQuery 1.4 o evento submit se propaga pelo efeito bolha (bubble) no Internet Explorer e é consistente nos navegadores modernos.

Notas adicionais:

  • Formulários e seus elementos não devem usar nomes ou ids iguais aos das propriedades do formulário, tais como submit, length ou method. Conflitos de nomes podem causar falha no funcionamento dos formulários. Para uma lista completa dos nomes a evitar bem como para verificar sua marcação consulte DOMLint.

Exemplos:

Example: Código para permitir o envio de um formulário somente se uma condição for satisfeita: 

<!DOCTYPE html>
<html>
<head>
  <style>
  p { margin:0; color:blue; }
  div,p { margin-left:10px; }
  span { color:red; }
  </style>
  <script src="http://code.jquery.com/jquery-git.js"></script>
</head>
<body>
  <p>Digite 'correto' para validar a entrada.</p>
  <form action="javascript:alert('success!');">
    <div>
      <input type="text" />
      <input type="submit" />
    </div>
  </form>
  <span></span>
<script>
    $("form").submit(function() {
      if ($("input:first").val() == "correto") {
        $("span").text("OK, entrada válida...").show();
        return true;
      }
      $("span").text("Entrada inválida!").show().fadeOut(1000);
      return false;
    });
</script>
</body>
</html>

Exemplo: Código para permitir o envio de um formulário somente se uma variável flag for definida: 

$("form").submit( function () {
  return this.variavel_flag;
} );

Exemplo: Código para disparar o evento submit para o primeiro formulário da página: 

$("form:first").submit();
